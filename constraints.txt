# constraints.txt — pinned, known-good Python dependency versions.
# See CHARTER.md "Standing decisions → Pinned toolchains".
#
# This is the Python analogue of Cargo.lock. The per-tier requirements files
# (hardware/requirements.txt, provision/requirements.txt) keep `>=` FLOORS so a
# platform without a matching wheel can still install the core; this file freezes
# the exact versions the test suite is verified green against. Install with:
#
#     pip install -r hardware/requirements.txt   -c constraints.txt
#     pip install -r provision/requirements.txt  -c constraints.txt
#
# A constraint only pins a version IF that package is being installed (directly or
# transitively); it never forces an install on its own. (deploy/update.sh passes
# this file on the Pi.)
#

# Test Coverage & Invariants Audit

## Health: 4 — Strong, well-disciplined suite; the fail-closed cores are exhaustively pinned, with a few real holes on cross-tier error paths and the header SSOT.

## Top findings (max 3, most important first)
1. **TEST-001 (P1):** The Pi camera-client's three evidence-POST fail-closed branches (edge 503/non-200, POST timeout/connection error, malformed-200 body) are NOT exercised. The existing fail-closed client tests only fail the *heartbeat* after a 200 `mitigate` — the edge-says-503/timeout-on-evidence handshake (the most important cross-tier fail-closed contract) is unpinned.
2. **TEST-002 (P1):** The shared `[headers]` SSOT is only half-enforced. Both Python senders (`camera_pusher.build_headers`, `client._request_headers`) HARDCODE the header strings instead of using the generated `contract_gen.HEADER_*` constants the Rust edge reads — and no test pins the wire headers against the spec. A rename in `spec.toml` would silently break the Pi→edge contract with all tests green.
3. **TEST-003 (P2):** The destructive cross-tenant delete guard (`sweep_evidence_days`: `k.starts_with("g/{gid}/evidence/")`) has no unit test. Its filter logic is buried in a host-I/O-bound private fn, so the "only ever delete under this garden's prefix" belt-and-suspenders claim is asserted nowhere.

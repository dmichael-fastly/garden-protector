# Shared Web Frontend (UI plumbing) Audit
## Health: 4 — Strong SSOT discipline (nav + ?v= single-sourced and test-gated); the notable gap is the History nav-active state being server-rendered wrong and patched in JS, plus the known per-page Pi-health duplication.

## Top findings (max 3, most important first)
1. **History route renders the wrong active nav link server-side on BOTH tiers** (UI-001). `/history` is served with `active_id="nav-dashboard"` baked in; `dashboard.html` JS then strips/re-adds `.active`. With JS disabled (or before hydration) History highlights "Dashboard" — the active-nav SSOT is effectively broken for that route, and the JS hand-rolls what `GP.markNav` already does.
2. **Pi-health CSS + SSE consumer duplicated across 5-6 pages in two divergent variants** (UI-002, UI-003). `.pi-health*` styles and the `/api/system/stream` consumer are copy-pasted; dashboard/devices/settings use a raw-EventSource `initPiHealthStream()` while storage/logs use `GP.sse`-based `pollSysInfo()`. This is exactly the deferred consolidation follow-up — fold style into gp.css and the consumer into a `GP.subscribeSystem`-style helper.
3. **Cache-busting + nav SSOT are genuinely well-built and test-gated** (positive). `?v=__ASSET_VERSION__` is centralized, stamped at serve time on all three tiers, and guarded by `tests/test_asset_version.py`; the nav model + ASSET_VERSION live only in `contract/spec.toml` and are rendered by the generated `render_nav` on both tiers. No drift found here.

## Cross-domain handoffs
- **UI-001 (edge expert):** the edge route table in `backend/src/main.rs:909` groups `/` `/admin` `/history` under one arm calling `dashboard_header_html()` with a fixed `nav-dashboard`. Fixing the History active state needs the edge handler to branch on path and pass `"nav-history"` (mirror the Pi change). Coordinate the small main.rs change with the edge owner; the existing cargo test at main.rs:1477-1485 asserts the current (Dashboard-active) behavior and would need updating.
- **UI-001 (pi expert):** `hardware/portal.py:1040-1043` likewise serves `/history` with `nav-dashboard`; the Python side needs the matching `"nav-history"` fix.
- **UI-002 / UI-003 (tests/ci expert):** if the Pi-health style/JS is consolidated into gp.css/gp.js, bump `[ui].asset_version` in `contract/spec.toml` + `make gen` (the cache-bust contract) — `tests/test_asset_version.py` and `tests/test_contract.py:57` gate this. Flag to CI that the consolidation PR must include the version bump or both tiers serve mismatched assets.
- **UI-004 (cloud/console expert):** the console login leaf emoji lives in `provision/console.py:106` (server-emitted HTML), so the swap touches console Python, not just static HTML.
- **No safety/security findings:** spray/mitigation fail-closed, CDN-first archive reads (all archive image fetches go through `/api/archive/image` proxy, never direct FOS), no-cloud-edge-logs, and inference-on-edge are backend concerns and were not contradicted by anything in the shared UI assets or pages; no secrets appear in any audited frontend file.

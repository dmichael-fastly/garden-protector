# UX & Accessibility Audit
## Health: 3 — Polished, coherent design language let down by motion/contrast accessibility gaps and a jarringly technical Logs page.

## Top findings (max 3, most important first)
1. **No `prefers-reduced-motion` support anywhere** (10 animations incl. pulsing safety pills, blinking deploy cursor, infinite spinners) — a WCAG 2.2 violation that affects every surface (UX-001).
2. **The Logs page breaks the product's voice entirely** — engineer jargon ("audit trail", "FSM State", "Correlation Trace", "Querying telemetry data...") plus native `confirm()`/`alert()` dialogs that ignore the theme and design language (UX-002, UX-003).
3. **User-visible copy leaks internal terms** the project explicitly bans — "the Pi", "edge registry in the cloud", "Fastly Secret Store", "Node ID", "ESP32" appear in dashboard hints and the Gadgets remove/edit modals (UX-004).

## Cross-domain handoffs
- **UX-002 / UX-003 / UX-004 (implementation) → ui-expert.** I set the copy direction and the modal-replacement pattern; ui-expert wires the exact strings into the pages and swaps the native dialogs for the shared `.modal` component. The copy itself is my call; the DOM/SSOT wiring is theirs.
- **UX-001 reduced-motion CSS → ui-expert.** The single `@media (prefers-reduced-motion)` block belongs in `gp.css`; I'm specifying it, ui-expert lands it (a gp.css change means bumping `asset_version` in contract/spec.toml + `make gen` — that asset-version/cache plumbing is ui-expert/edge territory, not mine).
- **UX-005 aria-live attributes → ui-expert.** Trivial markup additions to storage/costs/timelapse.
- **UX-006 console parity → ui-expert,** with a note to **edge/pi** if surfacing armed/spraying in the console detail needs any new field on `/api/console/state` (it already returns `armed`/`override_stop`/`continue_mitigation`, so likely none).
- **UX-007 "Stop Mitigation" relabel:** copy is mine, but confirm with **edge** that the relabel doesn't imply a behavior change to `apply_control` — it must remain the same persistent STOP override (no logic change), so this is label-only and fail-closed semantics are unchanged.
- No findings belong to **tests/ci/cloud** — the asset_version/spec.toml SSOT, CDN-first reads, no-cloud-edge-logs, and inference-on-edge policies were checked against the rendered surfaces and showed no UX-visible violation (e.g. archive images are fetched via `/api/archive/image` which the edge signs through the CDN read path, not direct FOS).
